# Customer Churn Prediction Analysis

This project performs an end-to-end data analysis and predictive modeling on a telecommunications customer dataset to identify factors leading to churn.

## 📊 Project Highlights
- **Data Cleaning:** Processed features like `TotalCharges` and handled missing values.
- **Exploratory Data Analysis (EDA):** Discovered that customers with **Month-to-month contracts** and those using **Electronic checks** have significantly higher churn rates.
- **Feature Engineering:** Created tenure groups and charge categories to improve model interpretability.
- **Machine Learning:** Implemented a Decision Tree Classifier to visualize the decision path for churn.

## 🛠️ Tech Stack
- **Language:** Python
- **Libraries:** Pandas, NumPy, Matplotlib, Seaborn, Scikit-Learn

## 📈 Key Findings
- Customers with a high number of technical tickets are more likely to leave.
- Higher monthly charges are correlated with increased churn risk.
